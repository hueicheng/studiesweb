Safari 是 Apple 公司所開發的瀏覽器，並內建於Mac OS X。

是 iPhone 與 iPod touch 的指定瀏覽器。

Safari 使用 Apple 自家的 WebKit 來進行網頁的排版及執行JavaScript。

Safari 是第一個支援 HTML 5 音訊與視訊標籤的瀏覽器。這些標籤讓開發人員得以將媒體直接整合到標準網頁中，進而減少反應時間並帶來更快的反應速度，其對 HTML 5 視訊的全螢幕與隱藏式字幕的支援，讓開發人員開發出更豐富且更方便取用的影片體驗。藉著對 HTML 5 離線技術的支援，Safari 允許以往業為基礎的應用程式將資料儲存在硬碟上，因此就算沒有連上 Internet 仍然可以使用。

Safari 也是第一個使用 CSS 3 動畫的瀏覽器，這項標準將網頁帶入新的層次。Safari 也首先支援 CSS 特效，藉著漸層、精確的遮罩與酷炫的映射效果將影像和照片加以風格化。

先來介紹下 Safari 瀏覽器的內容

先從 Safari 面板開始介紹，Safari 預設是將面板顯示只有上/下頁、網址列、搜尋列、功能選項及設定選項在上方

下方則是瀏覽紀錄、常用頁面及預設好的書籤，在下來才是分頁列

由於 Safari 並沒有向Google 和 Opera 一樣，將搜尋跟網址列結合所以在右方又多了一個搜尋列

Safari 預設的搜尋只有三個，Google、Yahoo、Bing，這三項可以說是台灣人常用的選項吧

雖然並不是每個人都慣用這三種搜尋引擎，畢竟其他的搜索引擎有其他的好，這點在這不討論

另外要如何增加其他搜索引擎筆者似乎也還未摸索到..

在下方的預設書籤裡，倒是有增加些比較常用到的書籤，例：Yahoo、Google地圖、維基百科、Youtube

在最旁邊還有新聞跟熱門兩個資料夾，裡面擺放的也都是國人常用的網站像Yahoo新聞、Google新聞、無名小站、104人力銀行

這些設定大概是在台灣版本的 Safari 才有的吧--(畢竟我不覺得外國人也是用無名來寫網誌= =--

這些書籤的設定可以說是比較貼心地設定，畢竟越貼心的服務才能招攬越多的客人，這點似乎在其他行業也是一樣的

**常用頁面**

Safari 將常用頁面像 Opera 一樣分常好幾格排列，但不一樣的是 Safari 的格式比較多也做成半圓弧狀看起來有立體感(?)

或許只是為了讓人看起來新穎的多吧，一樣可以使用 ctrl + 數字鍵作為鍵盤查看頁面

點擊之後還會有縮放的特效，不得不承認 Safari 是個很注重特效的瀏覽器

畢竟在這方面其他牌的瀏覽器沒有做到，在上方可以選擇常用的頁面或者是瀏覽紀錄

有別於常用頁面，瀏覽紀錄則是採用了單頁直立式的特效，讓使用者可以清楚的知道自己瀏覽過甚麼網頁

換頁的方式也是利用特效讓這瀏覽器看起來很特別，並且將上/下一個頁面放於旁邊，感覺上就像是在翻書一樣

在常用頁面下方還可以可以用來編輯的按鈕，可以固定常用網頁或是刪除不想看的網頁

其缺點就是不能手動輸入網址來新增，似乎只會從瀏覽紀錄中自動擷取，這點倒不是讓我很喜歡

畢竟瀏覽紀錄中有些只是看看的網頁而並非真的想當成用網頁來用，所以真要來設定可能要花上一些時間來摸縮

不過還是有方法直接輸入網址新增，也就是在常用頁面中的網址列先輸入網址再將其網址拖曳到下方

挺麻煩的就是了...

**書籤**

Safari 的書籤設置在左上方的書本按鈕，點選後並不是像其他瀏覽器一樣是下拉式選單

而是另外開個頁面將所有書籤顯示出來，在右上方有個畫面是顯示書籤的內容

其畫面也是跟瀏覽紀錄一樣慘用單頁直立式瀏覽，加入書籤的方式則是在碰到想要的網頁時在網址列的左方按下"+"

就可以將網頁加入為書籤。

**延伸功能**

Safari 有有類似於 FX 的外掛程式，在工具選項中選取喜好設定->延伸功能->取得延伸功能

會自動地轉跳到下載外掛程式的網頁，由於現在的使用者越來越要求方便、快速

而網站的功能卻遠遠不及使用者的需求，所以會有越來越多的外掛程式供使用者下載

由於FB 跟推特的使用者增加劇烈，所以外掛網站上關於FB跟推特的外掛還不少...

不過，這種安裝外掛的瀏覽器通常也會隨著外掛程式的增多而使的瀏覽器肥大

所以建議還是挑常用及適用的就好...

**Flash**

由於蘋果並不適很喜歡 Flash 這傢伙，所以大部分執行 Flash 的網站蘋果都有他的方式來另外執行

至於是甚麼方式筆者尚未摸清楚，只知道大概原理應該是用某種東西去繞過 Flash 在執行程式

這麼做的效果雖然慢了些許不過還是會達到使用的效果，不過對於筆者這種常玩Flash遊戲的人而言實在是有點受不了..

**其他**

Safari 跟 Opera 一樣，當你切換頁面的時候才會執行所需要的程式，否則就是一直停在那裏

這點來說主要是不希望瀏覽器一次大量工作而使得電腦的速度被拖累到

畢竟瀏覽器主要就是使使用者能夠高速的在網上衝浪，而不會碰到暗礁(lag)進而落水(當機)

不過這種東西難免會發生，目前筆者使用 Safari 倒是很少碰到就是

另外，在對於 HTML 5 方面，Safari 也擁有不錯的性能及支援通過[*http://www.html5test.com/ 這個網頁]的測試

可以看到 Safari 的支援度擁有著不錯的分數，目前各大瀏覽器都朝著 HTML 5 這個開發中的標準前進

不過比起 Chrome、Firefox、Opera這些瀏覽器，Safari 似乎還要在更加把勁

否則 Safari 可能會在名為 HTML 5 的瀏覽器戰爭中輸掉...
